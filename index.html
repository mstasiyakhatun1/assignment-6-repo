the difference between var, let, and const?
            var: Function-scoped, can be redeclared and reassigned, hoisted with undefined (avoid using).
            let: Block-scoped, cannot be redeclared but can be reassigned, hoisted but not initialized.
            const: Block-scoped, cannot be redeclared or reassigned, hoisted but not initialized, but objects/arrays can be
            modified.
            Best practice: Use const by default, let if reassignment is needed, and avoid var.
the difference between map(), forEach(), and filter()?
          map(): Returns a new array with modified values.  (Used for transformation)

          forEach():  Executes a function on each item, no return value.  (Used for looping)

          filter(): Returns a new array with elements that pass a condition.  (Used for filtering)
explain arrow functions and how they are different from regular functions
            Arrow Functions (=>) are a shorter way to write functions.
            Key Differences:
            this binding: Arrow functions inherit this from their surrounding scope, while regular functions have their own this.
            arguments object: Regular functions have it, arrow functions do not.
            Constructor (new) usage: Regular functions can be used as constructors, arrow functions cannot.
how JavaScript Promises work 
              JavaScript Promises handle asynchronous tasks.
              They have three states: Pending, Fulfilled (Success), and Rejected (Failure).
              Use .then() for success, .catch() for errors, and .finally() for cleanup.
              Common use cases: API calls, file reading, database operations.
how closures work in JavaScript 
                A closure is a function that remembers its lexical scope (the variables from the outer function) even after the outer
                function finishes. It's useful for data encapsulation and persistent state. Closures allow inner functions to access
                outer variables.
